### [P1] Non-null assertion on `shapeLead()` could mask bugs
**File:** `src/follow-up-api.ts:35`
**Issue:** Every success response uses `shapeLead(updated)!` with a non-null assertion. The `shapeLead` function returns `null` when given a falsy input. Pattern repeated on lines 35, 59, 111, and 135. The `!` operator hides a potential silent null in a JSON response.
**Suggestion:** Add an explicit null guard instead of `!`:
```typescript
const shaped = shapeLead(updated);
if (!shaped) {
  res.status(500).json({ error: "Failed to shape lead response" });
  return;
}
res.json({ success: true, lead: shaped } satisfies FollowUpActionResponse);
```

### [P1] `approveFollowUp` flow ambiguity — approved draft may never be sent
**File:** `src/leads.ts:382-418`
**Issue:** `approveFollowUp()` atomically claims `follow_up_status = 'pending'`, then clears the draft and schedules the next follow-up. The approved draft (visible on the dashboard) is discarded and a new one is generated by the scheduler. If "Approve" means "send this draft," the actual SMS send is missing from the approve handler.

### [P1] CSRF guard missing on existing `api.ts` POST routes
**File:** `src/api.ts:123`
**Issue:** The new `follow-up-api.ts` correctly applies `csrfGuard` to all POST routes, but existing POST routes in `api.ts` (`/api/leads/:id/approve`, `/api/leads/:id/edit`, `/api/leads/:id/outcome`, `/api/analyze`) do not have `csrfGuard`. With cookie-based auth, these are now vulnerable to CSRF attacks.
**Suggestion:** Apply `csrfGuard` to all state-changing routes in `api.ts`:
```typescript
router.use(csrfGuard);  // after sessionAuth
```

### [P2] Unsafe cast of `req.body` in snooze handler
**File:** `src/follow-up-api.ts:71`
**Issue:** `const { until } = req.body as SnoozeRequestBody;` casts the raw body without validation. If `req.body` is null, undefined, or a non-object, destructuring will throw an unhandled error.
**Suggestion:** Add a body-shape guard before destructuring:
```typescript
if (!req.body || typeof req.body !== "object") {
  res.status(400).json({ error: "Request body must be JSON" });
  return;
}
```
